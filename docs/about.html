Why this exists

Vibecoding is evolving. This toolkit helps you grow with it.

Yes, this is opinionated

Ghostty for the terminal, Starship for the prompt, Claude Code for AI pairing. Modern CLI tools like ripgrep and fzf. MCP servers for connecting Claude to Figma and documentation. These are the tools I've found work well together — not the only options, just a good starting point.

You can swap anything out later. This just saves you from spending a weekend researching what terminal to use.

Vibecoding started with something magical

Describing what you wanted and watching AI build it. No memorizing syntax, no documentation rabbit holes — just conversation turning into working code.

Then you wanted more

Run things locally. Test instantly. Iterate without waiting. Pair with Claude right in your project. Stop juggling fifteen browser tabs. The terminal is faster once you get past the initial learning curve.

But getting there feels like a big jump

Traditional coding education assumes you want to become a software engineer. Bootcamps, degrees, months of study. That's great if it's your path, but maybe you're here to design, create, and build with AI — not optimize algorithms.

Local development is useful for vibecoding, but the leap from browser to terminal can feel overwhelming.

So I made this

One command. Nice terminal, AI coding help, version control, useful tools. Everything configured to work together.

No researching which terminal to pick. No Reddit rabbit holes. No weekend spent configuring. Just a starting point so you can get back to building.

You'll learn as you go

Git makes sense when you're collaborating. Terminal shortcuts click when they save you time. You'll tweak things when they don't fit your workflow. It happens naturally, when you're ready.

Building in new ways

Vibecoding is different from traditional programming. You don't need to think like a programmer to build things anymore.

This toolkit exists because the transition to local development should feel natural — not like abandoning what makes vibecoding enjoyable.
