Frequently Asked Questions

Common questions about installation, usage, and troubleshooting.

The installer sets up: Homebrew (package manager), Xcode Command Line Tools, Ghostty (terminal), GitHub Desktop, Claude Code, Starship (prompt), Node.js, Bun, Git, GitHub CLI, and CLI utilities (ripgrep, fzf, eza, bat, jq). It also configures SSH keys, shell aliases, and MCP servers.

Everything is open source and widely used in the developer community. You can uninstall any component later.

Yes. The script is open source and you can review it on GitHub before running. It only installs standard developer tools from trusted sources (Homebrew, official repos). It creates backups before modifying your shell configuration and asks for confirmation before major steps.

The script requires your password for system-level installations (Homebrew, Xcode CLI Tools), which is normal for package managers.

10-20 minutes depending on your internet speed and whether you already have some components installed. The longest part is usually downloading and installing Xcode Command Line Tools (if not already present).

The script creates a backup of your .zshrc file before making changes. If something fails, you can restore from the backup in your home directory (look for .zshrc.backup with a timestamp).

Common issues: weak internet connection (restart the script), Xcode CLI Tools dialog not appearing (manually run xcode-select --install), permission errors (ensure you have admin access).

The installer is designed as an opinionated all-in-one setup. However, you can fork the GitHub repo and modify the install.sh script to skip components you don't want.

After installation, you can uninstall any tool using Homebrew: brew uninstall tool-name

Not required, but highly recommended. The installer can skip GitHub setup if you don't have an account. However, version control and collaboration are core to modern development workflows.

If you don't have a GitHub account, the installer will offer to open the signup page for you.

Use Homebrew to keep everything up to date:

brew update — Update Homebrew itself
brew upgrade — Update all installed packages
brew upgrade package-name — Update a specific package

Starship config: ~/.config/starship.toml
SSH keys: ~/.ssh/
Claude Code config: ~/.claude.json

No, this installer is macOS-specific. It uses Homebrew and macOS-specific installation methods. The individual tools (Claude Code, Ghostty, etc.) work on other platforms, but you'd need to install them manually.

To remove individual tools: brew uninstall package-name
To remove Homebrew entirely: /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/uninstall.sh)"

To restore your shell config: Replace ~/.zshrc with your backup file.

No. The installer checks what's already installed and skips those components. If you have Homebrew, Node, Git, or other tools already, they won't be reinstalled. The script only installs what's missing.

This saves you from making dozens of decisions and spending hours on setup. It provides working configurations for Ghostty and Starship, sets up SSH keys properly, configures useful shell aliases, and ensures everything works together.

Manual installation means researching each tool, figuring out configurations, and troubleshooting integration issues. This eliminates that friction.
